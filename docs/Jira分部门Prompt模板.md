# Jira 分部门 Prompt 模板

本文档按常见部门拆分 Jira Server 的使用模板，方便不同岗位直接复制使用。

覆盖部门：
- 研发
- 测试
- 产品
- 市场

建议默认遵循：
- 先分析、先展示
- 再确认、再执行

## 一、研发部门

### 1. 研发任务拆解

```text
请使用 $jira-server-spec-to-backlog 将下面的研发需求拆解为 Jira Server backlog。

项目 key：RDTASK
需求内容：
<粘贴需求或技术方案>

请完成：
1. 给出父项建议
2. 拆分研发实现任务
3. 如有需要，补充子任务
4. 标明建议使用的问题类型

执行方式：先展示拆解方案，等我确认后再创建 Jira。
```

### 2. 研发周报

```text
请使用 $jira-server-status-report 生成研发周报。

项目 key：RDTASK
时间范围：最近 7 天
汇报对象：研发负责人
输出风格：简洁摘要

请重点汇总：
1. 已完成研发任务
2. 当前进行中的研发任务
3. 当前阻塞项
4. 下周优先事项
```

### 3. 研发问题分诊

```text
请使用 $jira-server-triage-issue 对下面的问题进行分诊。

项目 key：HTEK
问题描述：
<粘贴报错、日志、复现步骤>

请输出：
1. 关键词提取
2. 相似 issue
3. 是否属于重复 / 回归 / 新问题
4. 如果需要新建，请建议标题和描述结构

先展示结果，不要直接创建 Jira。
```

## 二、测试部门

### 1. 测试问题提单

```text
请使用 $jira-server-triage-issue 对下面测试发现的问题进行分诊。

项目 key：HTEK
测试环境：
<填写环境>
问题描述：
<粘贴现象、步骤、结果、预期>

请完成：
1. 搜索 Jira 中是否已有相似问题
2. 判断是否重复或回归
3. 如果没有，建议新建为 `故障` 还是 `任务`
4. 输出适合 Jira 的标题和描述结构

先展示结果，不要直接创建。
```

### 2. 测试任务拆解

```text
请使用 $jira-server-spec-to-backlog 将下面测试工作拆解到 Jira。

项目 key：RDTASK
工作内容：
<粘贴测试计划、用例设计或验证任务>

请优先考虑：
1. 用例设计类内容归类为 `用例撰写任务`
2. 通用执行类工作归类为 `任务`
3. 如需要，再细分子任务

先输出拆解结果，等我确认后再创建。
```

### 3. 测试周报

```text
请使用 $jira-server-status-report 生成测试进度周报。

项目 key：HTEK
时间范围：最近 7 天
汇报对象：测试负责人和项目经理
输出风格：详细版

请重点关注：
1. 已关闭问题
2. 当前未关闭高优先级问题
3. 回归风险
4. 当前测试阻塞
```

## 三、产品部门

### 1. PRD 拆解

```text
请使用 $jira-server-spec-to-backlog 将下面 PRD 拆解为 Jira backlog。

项目 key：HTEK
PRD 内容：
<粘贴 PRD 摘要或需求说明>

请完成：
1. 识别父项
2. 拆成需求 / 任务 / 子任务
3. 标明每项建议的问题类型
4. 如项目有必填字段，请提醒我

先展示结果，不要直接创建。
```

### 2. 会议纪要转任务

```text
请使用 $jira-server-meeting-to-tasks 将下面产品会议纪要整理成 Jira Server 任务。

项目 key：TASK
会议纪要：
<粘贴纪要>

请完成：
1. 提取行动项
2. 区分需求类事项和协调类事项
3. 建议负责人
4. 展示拟创建任务列表

等我确认后再创建 Jira。
```

### 3. 产品进度同步

```text
请使用 $jira-server-status-report 生成一份产品需求推进情况汇总。

项目 key：HTEK
时间范围：最近 14 天
汇报对象：产品负责人和项目经理
输出风格：简洁摘要

请重点汇总：
1. 已完成需求
2. 当前处理中需求
3. 影响排期的阻塞项
4. 近期需要确认的事项
```
