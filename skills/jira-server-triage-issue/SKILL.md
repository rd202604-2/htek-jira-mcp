# Jira Server 问题分诊

## 概述

对 bug 或错误报告进行分诊：提取故障特征，在本地 Jira Server 中搜索重复项，再决定是给已有问题补充评论，还是创建新问题。

## 工作流程

1. 提取可搜索信号。  
重点提取：
- 报错文本
- 组件或功能区域
- 环境信息
- 用户可见症状

2. 从多个角度搜索 `search_jira_issues`。  
建议分别使用以下维度构造 JQL：
- 完整或部分错误文本
- 组件关键词
- 症状关键词
- 最近未解决的问题

3. 用 `get_jira_issue` 检查最佳候选结果。  
如果最近的讨论内容有帮助，再补充使用 `get_jira_comments`。

4. 对搜索结果做分类：
- 高概率重复问题
- 有关但不完全相同
- 疑似回归问题
- 高概率新问题

5. 在修改 Jira 前先向用户展示结论。  
建议动作一般包括：
- 给已有 issue 加评论
- 新建一个 issue
- 新建 issue 并引用已有 issue

6. 根据用户选择执行动作。  
使用：
- `add_jira_comment`
- `create_jira_issue`
- 如果需要补字段，可用 `update_jira_issue`

## 创建问题建议

问题类型从项目元数据中选择：
- 优先使用 `故障`
- 如果项目没有 bug 专用类型，再回退到 `任务`

创建前应先调用 `get_jira_issue_type_fields` 检查必填字段。

建议概要格式：

```text
[组件] [故障] - 简短症状
```

示例：

```text
[音频] Whisper 导出失败 - 转写结果为空
```

## 评论建议

给已有 issue 补充证据时，尽量包含：
- 观察到的现象
- 环境信息
- 影响范围
- 复现线索

内容保持简洁、客观。

## 注意事项

- 不要无提示地创建重复 bug。
- 优先把最相似的候选问题展示给用户。
- 对已解决的历史问题，应优先视作“可能回归”，而不是直接认定为重复。
